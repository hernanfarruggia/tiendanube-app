## Notes

### Architecture Decisions
- **No local product storage**: Only `product_id` + `embeddings` stored. Product data fetched real-time from Tiendanube API when recommending
- **Why**: Always fresh data (price, stock, images), simpler architecture, minimal storage
- **Trade-off**: 3-5 API calls to Tiendanube per recommendation (mitigated with Promise.all parallel calls)

### Bugs Fixed Today
1. **Template bug**: HTTP header "Authentication" should be "Authorization" (affects ALL Tiendanube API calls)
2. **Template bug**: Missing `await` on userRepository.save() caused race condition with webhooks
3. **OAuth compliance**: Must redirect to store's `original_domain`, not generic tiendanube.com

### Critical Endpoints
**Public (no auth):**
- POST /api/chat - Storefront chat
- POST /api/webhooks/product/* - Tiendanube webhooks

**Authenticated (JWT):**
- POST /api/products/sync - Manual sync
- GET /api/products/placeholders - Typewriting suggestions
- GET /api/settings - Get customizations
- PUT /api/settings - Update customizations
